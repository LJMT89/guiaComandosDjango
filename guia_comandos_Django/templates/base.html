COMANDOS DJANGO

Entorno virtual

Crear entorno virtual

virtualenv nombreEntorno --python=python3

Activar entorno virtual

source bin/activate

Esto debe hacerse dentro de la carpeta del entorno virtual.

Instalar version específica de Django

pip install Django==3.2.13

Para mirar la versión instalada:

python -m django --version

Crear proyecto Django

django-admin.py startproject "nombre del proyecto" --> sin comillas

NOTA: Dentro de un entorno virtual:

django-admin startproject "nombre del proyecto" --> sin comillas y sin el .py

De acuerdo a las convenciones de Django, se recomienda utilizar minúsculas y guiones bajos para el nombre del proyecto. Por ejemplo, si el proyecto se llama "MiProyecto", se recomienda nombrarlo como "mi_proyecto".

Crear aplicación Django

django-admin.py startapp "nombre de la aplicacion" --> sin comillas

NOTA: Dentro de un entorno virtual:

django-admin startapp "nombre del proyecto" --> sin comillas y sin el .py

Migraciones

python manage.py makemigrations

python manage.py migrate

python manage.py collectstatic

Crear superusuario

python manage.py createsuperuser

Lanzar servidor

python manage.py runserver

Para elegir el puerto:

python manage.py runserver 0.0.0.0:8000

Requirements

Guardar

pip freeze > requirements.txt

Instalar

pip install -r requirements.txt

Matar procesos

Mata todos los procesos asociados al puerto 8000

sudo fuser -k 8000/tcp

Subir cambios al servidor

  1. Primero me logueo como usuario propietario

    Con el comando:

    sudo su - "propietario"

    Ejemplo: sudo su - rmuser

  2. Activar el entorno virtual

  3. Regresar nuevamente a la carpeta del proyecto y correr el comando collectstatic

    Nota: Si hay q correr migraciones se corren con el entorno activado

  4. Desactivo el entorno virtual --> deactivate
  5. Desloguear el usuario propietario --> exit
  6. Reiniciar la aplicación
  7. Reiniciar el servidor

Subir cambios al servidor desde GIT

  1. Revisar en la rama release:

    manage.py --> que esten apuntando a production

    wsgy.py --> que esten apuntando a production

    .env --> DEBUG = false y la BBDD

    settings/production --> BBDD, ALLOWED_HOSTS

  2. Abrir 2 consolas: una para ejecutar los comando del usuario de la aplicacion y otra para ejecutar los comandos del usuario de la VPS que siempre es ideafix.

  3. Me logueo como usuario propietario

    Con el comando:

    sudo su - "propietario"

    Ejemplo: sudo su - rmuser

  4. Ingresar a la carpeta del proyecto
  5. Ejecutar --> git status
  6. Ejecutar --> git pull
  7. Activar el entorno virtual (en la misma consola) e ir a la carpeta del proyecto
  8. Ejecutar --> python manage.py check
  9. Ejecutar --> migraciones
  10. Ejecutar --> collectstatic
  11. Entro a la segunda consola y ejecuto los comandos de reiniciar la aplicacion y reiniciar el servidor
  12. Ejecutar --> python manage.py check

Instalar ckeditor

A continuación se describe el proceso para instalar la librería ckeditor la cual admite campos tipo RichTextField en el modelo de Django.

  1. pip install django-ckeditor

  2. Agregar ckeditor en INSTALLED_APPS en el settings del proyecto.

  3. Ejecutar el comando collectstatic.

    4. Del aquí en adelante únicamente si se usa el tipo de campo ckeditor_uploader, que es para subir archivos adjuntos como imagenes.

  4. Agregar ckeditor_uploader en INSTALLED_APPS en el settings del proyecto.

  5. Agregar CKEDITOR_UPLOAD_PATH = "uploads/" en el settings del proyecto, ckeditor_uploader trabaja con la carpeta media.

  6. Agregar la ruta: path('ckeditor/', include('ckeditor_uploader.urls'))

A continuación se describe la forma de uso:
